# PRD — Graph RAG ATS (Applicant Tracking System)
**Version:** 1.0  
**Status:** Ready to build  
**Stack:** FastAPI · Neo4j AuraDB · Supabase · LangChain · Gemini · Celery · Redis · Google Stitch

---

## 1. Problem Statement

Traditional ATS systems reject qualified candidates because they match on exact keywords. A candidate who lists "ReactJS" may be rejected for a role requiring "React" or "frontend development." Prior systems cannot understand that Vue.js knowledge implies JavaScript proficiency, or that a "Data Analyst" role heavily overlaps with "Business Intelligence Analyst" skills.

This system solves that with a **Graph RAG (Retrieval-Augmented Generation) pipeline** backed by the **ESCO taxonomy** — an EU-standard ontology of ~3,000 occupations and ~14,000 skills with explicit relationships — enabling context-aware, semantically intelligent candidate shortlisting.

## 4. Key Workflows

### 4.1 Job Posting Workflow
```
HR logs in → fills job form (title, description, requirements, ESCO code)
→ sets eligibility filters (branch, year, gender, CGPA)
→ sets max applicants + deadline
→ publishes → job appears on candidate portal
```

### 4.2 Candidate Application Workflow
```
Candidate browses jobs → clicks Apply
→ fills basic form (name, email, phone, branch, graduation year, CGPA, gender)
→ uploads resume PDF
→ system runs eligibility check synchronously
  → FAIL: immediate rejection message, not stored for RAG
  → PASS: stored in DB, resume saved to Supabase Storage
→ candidate sees "Application submitted"
```

### 4.3 Shortlisting Workflow
```
HR opens job dashboard → sees applicant count
→ enters shortlist number (e.g. 20)
→ clicks "Run Shortlisting"
→ Celery task queued → HR sees progress bar (WebSocket)
→ For each eligible application:
    1. Resume PDF fetched from Supabase Storage
    2. Text extracted via PyMuPDF / Unstructured
    3. Skills/entities extracted from text
    4. Each skill mapped to ESCO nodes via Neo4j
    5. Graph traversal scores candidate vs job's ESCO occupation
    6. Score calculated: direct skill match + 1-hop related skills + 2-hop adjacent
    7. Gemini generates explanation paragraph
→ Top N candidates marked "shortlisted", rest "rejected"
→ Emails sent via Resend:
    - Shortlisted: congratulations + explanation of selection
    - Rejected: regret + specific skill gap suggestions from Gemini
→ HR dashboard updates with ranked shortlist
```

## 5. Graph RAG Scoring Model

### Node Types in Neo4j
```
(:Occupation {escoCode, label, description})
(:Skill {escoCode, label, skillType, description})
```

### Relationship Types
```
(:Occupation)-[:REQUIRES {importance: 'essential'|'optional'}]->(:Skill)
(:Skill)-[:BROADER]->(:Skill)          # skill hierarchy
(:Skill)-[:RELATED_TO]->(:Skill)       # cross-skill relations
(:Candidate)-[:HAS_SKILL]->(:Skill)    # created during scoring
```

### Scoring Formula
```
score = (direct_matches × 3) + (1_hop_matches × 1.5) + (2_hop_matches × 0.75)
normalized_score = score / max_possible_score × 100
```

Where:
- **direct_matches**: candidate skill found directly in job's ESCO skill set
- **1_hop_matches**: candidate skill is BROADER/RELATED_TO a required skill
- **2_hop_matches**: candidate skill is 2 hops away in the graph

## 6. Email Templates

### Shortlisted Email
- Subject: `You've been shortlisted for [Job Title] at [Company]`
- Body: Generated by Gemini with: candidate name, job title, specific skills matched, next steps

### Rejected Email
- Subject: `Your application for [Job Title]`
- Body: Generated by Gemini with: empathetic tone, specific skill gaps identified, suggested learning resources from ESCO

## 7. API Endpoints (FastAPI)

| Method | Endpoint | Auth | Description |
|---|---|---|---|
| POST | `/auth/register` | Public | Register recruiter or candidate |
| POST | `/auth/login` | Public | Login, returns JWT |
| GET | `/jobs` | Public | List active jobs |
| POST | `/jobs` | Recruiter | Create job post |

## 8. Non-Functional Requirements

| Requirement | Target |
|---|---|
| Resume analysis time | < 5 seconds per resume |
| Shortlisting 100 resumes | < 10 minutes total |
| API response time | < 300ms for non-RAG endpoints |
| Resume file size limit | 5MB PDF only |
| Concurrent shortlist jobs | 1 per job (queue prevents duplicates) |
| Email delivery | Within 2 minutes of shortlisting completion |
